# 交易策略与组合配置规范

> 本文档是策略生成、资金分配、风险控制的**单一规范来源**。
> 所有数值均为建议默认值,落地后全部进 config/数据库,可改不改代码。
> 状态:**规划中,未实施** — 落地映射见第 5 节,分三期。

---

## 0. 层次总览

自上而下四层,每层只管自己的事:

```
组合层  总风险预算、组合级熔断(总开关)          → config
  └─ 桶层  风险份额、桶熔断、风格约束            → buckets 表
      └─ 策略层  模板×参数×品种×周期 = 实例, 带桶tag → strategies 表
          └─ 订单层  等风险手数、服务端SL/TP        → runner 计算
```

- **上层管预算,下层管执行**:桶不关心某笔单,策略不关心别的策略,订单只执行公式
- **钱的语言统一用"风险"**:分配的是"允许亏多少"(风险预算),不是手数、不是本金
- 每个策略实例必须归属且只归属一个桶(生成时指定,不允许无桶策略)

## 1. 桶(风险预算分层)

### 1.1 桶定义(4 桶,可增减)

| 桶 key | 名称 | 定位 | 风险份额 | 单笔风险 | 周期 | RR 范围 | 预期频率 |
|--------|------|------|---------|---------|------|---------|---------|
| `core` | 稳健核心 | 靠数量和分散出平滑曲线,主力收益来源 | **50%** | 0.25% | M15~H1 | 1.5~2.5 | 高(策略多×多品种) |
| `aggr` | 激进卫星 | 高赔率低胜率,博大行情 | **25%** | 0.50% | H1~H4 | 3.0~4.0 | 低,连亏是常态 |
| `macro` | 长线趋势 | 大周期趋势,持仓数天~数周,与日内低相关 | **15%** | 0.40% | H4~D1 | 2.5+ 或跟踪止损 | 很低(月数笔) |
| `probe` | 实验仓 | 新模板/AI生成/MQ5翻译的首次实弹验证 | **10%** | 0.10% | 不限 | 不限 | 不限 |

设计理由:
- `core` 份额最大但单笔最小——它的优势来自分散,不来自单笔赌注
- `aggr` 单笔最大但份额受限——博大钱的仓位天然要被隔离,亏完预算就停
- `macro` 提供与日内策略低相关的第三条收益流(分散的主要价值)
- `probe` 是策略工厂特有的:任何新东西先用最小风险跑真钱,数据达标才准进其他桶

### 1.2 每桶配置字段(buckets 表)

| 字段 | 含义 | core 默认 | aggr 默认 | macro 默认 | probe 默认 |
|------|------|-----------|-----------|------------|------------|
| `risk_share` | 组合风险份额 | 0.50 | 0.25 | 0.15 | 0.10 |
| `per_trade_risk_pct` | 单笔风险(占总净值) | 0.25% | 0.50% | 0.40% | 0.10% |
| `dd_pause_pct` | 桶熔断线(桶累计回撤占总净值) | 6% | 5% | 4% | 1.5% |
| `max_positions` | 桶内最大同时持仓数 | 10 | 4 | 4 | 3 |
| `paused` | 暂停开新仓(手动或熔断触发) | false | false | false | false |

### 1.3 暂停与熔断(三级,只停开仓不砍持仓)

**原则:暂停 = 不开新仓;已有持仓靠服务端 SL/TP 自然了结(铁律 2 已保证),不主动平仓。**

| 级别 | 触发 | 动作 | 恢复 |
|------|------|------|------|
| 策略级 | 单策略连亏 N 笔(默认 8)或回撤超其桶单笔风险×10 | 该策略回 ARCHIVED | 人工复核,重走漏斗 |
| 桶级 | 桶累计回撤 ≥ `dd_pause_pct` × 总净值 | 桶 `paused=true`,桶内全部策略停止开仓 | 人工在 web 上恢复(不自动恢复,强制复盘) |
| 组合级 | 总净值回撤 ≥ 15%(config: `portfolio_max_dd`) | 全部桶暂停(总开关) | 人工 |

- 手动暂停:web 上每桶一个暂停/恢复按钮,与自动熔断共用同一个 `paused` 字段
- runner 每次开仓前检查:策略所属桶 `paused` → 拒单并记日志(实现上 runner 拉策略清单时 api 直接过滤掉暂停桶的策略,双保险)
- 自动熔断依赖成交回写数据库(P3),之前只有手动暂停(P1 即可用)

### 1.4 组合层配置(config 表)

```
portfolio_max_dd:        0.15    # 组合级熔断
per_symbol_max_notional: 2.0     # 同品种名义价值上限(倍数,相对净值)
min_volume:              0.01    # 手数下限(券商约束)
```

## 2. 策略类型路线图(按桶)

### 2.1 现有策略归桶

| 模板 | 现状 | 归桶建议 |
|------|------|---------|
| `ma_cross` | 20 个 XAUUSD M15 实例在 demo | rr<2.5 → `core`;rr≥3 → `aggr`;**同品种同模板相关性≈1,core 桶真正的分散要靠 2.2 补品种和模板** |
| `breakout` | 已有模板,未批量生成 | channel≤55 → `core`;channel≥100 且 rr≥3 → `aggr` |

### 2.2 建议新增模板(按桶补内容,每个桶至少 2 个不相关模板)

**core(稳健核心)——补"均值回归"类,与趋势类天然负相关:**

| 模板 | 逻辑 | 核心参数 |
|------|------|---------|
| `bb_revert` | 布林带均值回归:触下轨+RSI超卖做多,回中轨止盈 | `period` 14~30, `dev` 2.0~3.0, `rsi_th` 20~35 |
| `rsi_revert` | RSI 极值反转,横盘市专用 | `period` 7~21, `os/ob` 20/80~30/70, `rr` 1.0~1.5 |
| `ma_cross` 多品种 | 现有模板铺到 EURUSD/GBPUSD/USDJPY | 现有参数空间 |

**aggr(激进卫星)——补"大周期突破"类:**

| 模板 | 逻辑 | 核心参数 |
|------|------|---------|
| `donchian_break` | 唐奇安通道突破(海龟式),H1~H4 | `channel` 20~100, `rr` 3~4 |
| `momentum` | ROC 动量:N 根涨幅超阈值追入 | `lookback` 10~40, `threshold` |

**macro(长线)——补"日线趋势跟随":**

| 模板 | 逻辑 | 核心参数 |
|------|------|---------|
| `trend_follow_d1` | D1 长均线方向过滤 + H4 回调入场,ATR 跟踪止损 | `ma` 100~200, `atr_k` 2~3(跟踪) |
| `weekly_break` | 周高低点突破,持仓数周 | `lookback_weeks` 4~12 |

> 跟踪止损需要 runner 支持改单(order_modify),是 macro 桶的前置开发项。

**probe(实验)——不新增模板,承接:** AI 生成器产物、MQ5 翻译产物、任何新模板的第一批实例。

### 2.3 参数规范:止损统一改用 ATR(重要升级)

现在的 `sl_points` 固定点数有两个硬伤:跨品种没有可比性(150 点对 EURUSD 和 XAUUSD 完全两码事);不随波动率适应(平静期止损太远、狂暴期太近)。

**规范:新模板一律 ATR 止损,现有模板加 ATR 变体:**

```
SL 距离 = atr_k × ATR(atr_period)     # atr_k ∈ [1.5, 3.0], atr_period 默认 14
TP 距离 = rr × SL 距离                 # 或跟踪止损(macro 桶)
边界保护: SL 距离 clamp 到 [券商 stops_level×2, 净值风险可承受上限]
```

参数空间从 `sl_points: (100,1500)` 变为 `atr_k: (1.5, 3.0, 0.25)`——搜索空间更小、跨品种通用、天然抗波动率变化。

### 2.4 生成规则(强制)

1. **生成时必带 `bucket`**:web 生成表单选桶;API `generate` 请求 bucket 为必填;无桶不入库
2. 生成的参数必须落在桶约束内(周期、RR 范围与 1.1 表一致),API 校验
3. 反过拟合门禁不变(样本外/参数邻域/最少笔数),按桶可以有不同阈值(probe 最宽松)
4. 建议自动 tag:`template`、`generated_by`(random/grid/ai/mq5)进 tags 字段,便于日后归因

## 3. 订单层规范

### 3.1 手数公式(等风险)

```
单笔风险额 = 账户净值 × 桶.per_trade_risk_pct
手数 = 单笔风险额 ÷ (SL点数 × 每点每手价值)     # 向下取整到 0.01
```

数例(净值 $100,000):

| 桶 | 风险额 | 策略 SL | 每 0.01 手每点 | 手数 |
|----|--------|--------|---------------|------|
| core 0.25% | $250 | XAUUSD 1000 点 | $0.01 | **0.25 手** |
| aggr 0.50% | $500 | XAUUSD 300 点 | $0.01 | **1.66 手** |
| probe 0.10% | $100 | XAUUSD 1000 点 | $0.01 | **0.10 手** |

对比现状:全员固定 0.01 手,#188(SL 150 点)单笔风险 $1.5、#199(SL 1500 点)$15——相差 10 倍是参数碰巧决定的。等风险公式让每笔亏损上限一致,权重回到人手里。

### 3.2 止损止盈(铁律不变 + 增强)

- 每笔订单**必带服务端 SL/TP**(现有铁律,runner 已强制)
- SL 按 2.3 的 ATR 规范;TP = rr × SL;macro 桶用跟踪止损(P2 开发)
- 下限保护:SL 距离 ≥ 券商 `stops_level` × 2,否则拒单

### 3.3 敞口上限(逐单前置检查,任一超限拒单)

1. 策略级:同策略同时最多 1 仓(现状保持)
2. 桶级:桶内持仓数 ≤ `max_positions`
3. 品种级:同品种总名义价值 ≤ 净值 × `per_symbol_max_notional`(防 20 个金策略同向叠满)
4. 组合级:保证金水平必须 > 追保线 × 3

## 4. 关于杠杆(一句话定位)

杠杆是**输出不是输入**:以上规则(单笔风险、桶份额、敞口上限)共同决定总敞口,券商杠杆(1:100)只是保证金约束。永远不"选杠杆",只定风险预算。

## 5. 数据库与系统落地映射

### 5.1 新表 `buckets`

```sql
CREATE TABLE buckets (
    key                VARCHAR(16) PRIMARY KEY,          -- core/aggr/macro/probe
    name               VARCHAR(32) NOT NULL,
    risk_share         NUMERIC(4,2) NOT NULL,            -- 0.50
    per_trade_risk_pct NUMERIC(5,4) NOT NULL,            -- 0.0025
    dd_pause_pct       NUMERIC(4,3) NOT NULL,            -- 0.06
    max_positions      INTEGER      NOT NULL,
    paused             BOOLEAN      NOT NULL DEFAULT FALSE,
    notes              TEXT,
    updated_at         TIMESTAMPTZ  NOT NULL DEFAULT now()
);
```

### 5.2 strategies 表加列

```sql
ALTER TABLE strategies ADD COLUMN bucket VARCHAR(16) REFERENCES buckets(key);
ALTER TABLE strategies ADD COLUMN tags   JSONB NOT NULL DEFAULT '[]';
-- 迁移: 现有策略回填 (rr<2.5→core, rr≥3→aggr), 之后 bucket 设 NOT NULL
```

### 5.3 各服务改动

| 服务 | 改动 |
|------|------|
| api | `generate` 必填 bucket + 桶约束校验;`/strategies/status` 返回 bucket 且过滤暂停桶;新增 `/buckets` CRUD + 暂停/恢复 |
| web | 生成表单选桶;策略/Demo/Live 页显示桶列;新"组合"面板:每桶份额/持仓/盈亏/暂停按钮 |
| runner | 拉取时带桶配置;开仓前跑 3.1 手数公式 + 3.3 敞口检查;`VOLUME` env 降级为 fallback |
| strategy_core | 新模板按 2.2 逐个加;ATR 止损基类支持 |

### 5.4 分期

| 期 | 内容 | 前置 |
|----|------|------|
| **P1** | buckets 表 + strategies.bucket/tags + 生成带桶 + web 展示与手动暂停 | 无,随时可做 |
| **P2** | runner 等风险手数 + 敞口检查 + ATR 止损模板 + 新模板(core 优先) | P1 |
| **P3** | 成交回写数据库 → 自动熔断(桶/策略级)+ 跟踪止损(macro) | P2 + 对账链路 |
