# v1.3 回测质量(反过拟合)

> 状态:**跨品种验证 + 三层显示 已实现**(schema 008 / api / web);OOS 时间切分等见 §1 仍为规划。
> 目标:根治"回测显示很好、demo 完全不一样"——本质是**过拟合**(在短历史上挑中了靠运气的漂亮参数)。
> 边界:**只加回测的"筛选质量",不碰撮合引擎**(撮合已足够悲观,见下)。策略逻辑、下单路径一律不动。

---

## 0. 先厘清:撮合是准的,漏的是"筛选"

`services/backtest.py` 的撮合已经足够悲观、足够真实,**不是问题所在**:

- 信号在 TF bar 收盘产生 → **下一根 M1 开盘价成交**(不偷看未来)
- 进场按 ask+点差+滑点,离场按 bid;**SL 先于 TP 检查**(同 bar 悲观)
- **跳空按实际开盘价成交**(sl_gap/tp_gap),不按挂单价
- 佣金按点数逐笔扣除;点差默认用每根 bar 记录的真实点差

所以"回测好 demo 差"**不是撮合失真**,而是:**没有任何反过拟合的筛选**——随机生成的一批参数里,总有几条在这段历史上碰巧漂亮;拿去 demo(=样本外)就现原形。

**结论:要加的是"筛选质量"(反过拟合),不是改撮合。**

---

## 1. 反过拟合总方案(手段 / 治什么 / 代码改动风险 / 是否做)

过拟合有两个正交维度,分别对应两根柱子:

```
时间维度(#1 OOS):     同品种, 过去 vs 未来(留出段)都得赚
空间维度(#5 跨品种):   同参数, 这个品种 vs 别的品种都得赚
两个都过 = 才够格进 demo
```

| # | 手段 | 治什么 | 代码改动 / 风险 | 是否做 |
|---|------|--------|----------------|--------|
| **5** | **跨品种验证**(空间维度) | 品种特异性过拟合(只在 XAU 历史巧合赚) | 同参数循环所有品种各回测一次,每品种存一行(backtests 加 symbol/broker 列)。**不碰撮合**,只是多跑几遍 + 聚合展示 | ✅ **已实现** |
| **1** | **样本外切分**(时间维度) | 过拟合主因(demo 就是样本外) | 回测跑完后按时间把 trades 切训练段/留出段,各算 metrics;排名先按训练段选、再要求留出段也赚。**纯后处理,撮合零改动** | 🔜 下一期(第一优先) |
| **2** | **分段最少笔数** | 小样本假象 | 训练段、留出段各自都要够笔数;`/backtest/top` 筛选加强。极低风险 | 🔜 跟 #1 一起 |
| **3** | **短样本警告** | 数据太浅放大一切过拟合(现仅~3个月) | 回测算 M1 时间跨度,不足门槛标"样本不足,结论不可靠"。低风险 | 🔜 跟 #1 一起 |
| **4** | **参数邻域稳健性** | 参数卡在历史噪声缝里(孤峰) | 对候选生成 ±10~20% 邻居批量回测,要求邻居也普遍赚。新编排,算量 ×N | 🔶 第二期 |
| **6** | **walk-forward** | OOS 的滚动升级(金标准) | 多窗口编排,大改 | ❌ 暂不做,OOS 单切够用 |

**数据深度不改代码,却是地基**:引擎不关心 3 个月还是 10 年,但样本越深、OOS 与跨品种越可信。深度是"下载/存储"的事(historical_bars 分区、券商能给多少),与回测逻辑无关。

## 2. 本期实现:全币种 / 全券商(opt-in 勾选)

回测运行页两个复选框,**默认不勾 = 现状(单品种,快)**:

```
批量回测  [状态] [品种] [上限] [成本…]   ☑ 跨全部货币对验证   ☐ 跨全部券商验证(暂不可用)   [启动]
```

### 全货币对验证 ✅ 可用
勾了 = **同参数在所有 `download=TRUE` 的品种(交易品种 + 验证品种)上各回测一次**。
- 运行时间 ×N(N=品种数);数据大了再加"只测验证品种"之类的过滤,现在先全测。
- 每品种一行结果,带上该品种的 `point` 与 `broker` 标注。

### 全券商验证 ⚠ 机制占位,当前不可用
**诚实说明**:`historical_bars` 主键是 `(symbol, timeframe, time)`,**没有券商维度** → 库里每个品种只有一条价格序列。要"同一品种跨多个券商比"需要**重量版 schema**(broker 进主键),那是以后的事。
- 所以复选框**禁用 + 提示**,不是能勾但没反应的假开关。
- 但结果已经带 `broker` 标注(来自 symbols 表),等真接了第二个券商、上了重量版 schema,显示会自动多出券商行——**机制先建好**。

## 3. 显示颗粒度(平衡:不炸 + 看得出适配品类)

痛点:`36 (11胜/25负)` 一个数看不出适配哪个品类;每品种拆一行又炸表。
**平衡点:主行保持一行 + 健壮性摘要列 + 点开看细分**,复用 worker 已有的"展开明细行"组件。

```
结果排名(每策略一行, 不炸)
┌────┬─────────┬──────┬──────┬──────┬─────┬────────┬──────────────┬──────┐
│ #  │ 策略     │ 品种 │ 券商 │ …    │ PF  │ 最大回撤│ 跨品种健壮性  │ 操作 │
├────┼─────────┼──────┼──────┼──────┼─────┼────────┼──────────────┼──────┤
│ 1  │ ma_cross│ XAU  │ ICM  │ …    │ 1.4 │ 820    │ ▾ 4/5 赚 🟢  │ …    │  ← 一眼看健壮
└────┴─────────┴──────┴──────┴──────┴─────┴────────┴──────────────┴──────┘
     点 ▾ 展开 ↓ (复用 subtable)
     ┌────────┬──────┬─────────┬──────┬─────┬────────┐
     │ 品种    │ 券商 │ 笔(胜/负)│ 净点数│ PF  │ 最大回撤│
     ├────────┼──────┼─────────┼──────┼─────┼────────┤
     │ XAUUSD主│ ICM  │ 36(11/25)│ +820 │ 1.4 │ 820    │
     │ GBPUSD │ ICM  │ 31(18/13)│ +510 │ 1.3 │ …      │
     │ EURUSD │ ICM  │ 28(15/13)│ +340 │ 1.2 │ …      │
     │ AUDUSD │ ICM  │ 22(14/8) │ +290 │ 1.1 │ …      │
     │ USDJPY │ ICM  │ 24(9/15) │ -180 │ 0.8 │ …      │ ← 这个品种不适配
     └────────┴──────┴─────────┴──────┴─────┴────────┘
```

**三层各管各的:**

| 层 | 显示什么 | 回答什么 | 颗粒度 |
|----|---------|---------|--------|
| 主行·排名列 | 主品种成绩(现状不动) | 这实例真去 demo/live 会怎样 | 粗,不炸 |
| 主行·健壮性列(新) | `4/5 赚 🟢`(几个品种里几个盈利) | 普适规律 还是 巧合 | 一眼摘要 |
| 展开明细(点开) | 每品种/每券商一行 | **适配哪个品类** / 哪个不行 | 细,按需 |

**关键:主行不拆、不改**——它是该实例真正要跑 demo/live 的主品种,最该显眼。"看不出适配品类"交给健壮性列 + 展开解决,不污染主行。

**健壮性徽章配色**(仅当跨品种测过、tested≥2 才显示,否则"未跨品种"):
- `5/5 🟢` 全赚 = 普适,最可信
- `≥60% 🟡` 多数赚 = 有适配性
- `<60% ⚪` 少数赚 = 要么真适配那品种、要么过拟合 → 配合 #1 时间 OOS 再判

## 4. 落地映射(本期 · 已实现)

| 层 | 改动 |
|----|------|
| schema | `008_backtests_symbol.sql`:backtests 加 `symbol` / `broker` 列(记录该行跑在哪个品种/券商数据上),存量行回填为各自策略主品种 + 索引 |
| api | `BacktestRequest` 加 `cross_symbol`;`_run_batch` 改按品种分组的外层循环(每品种 M1 只加载一次,内层跑所有策略),INSERT 带 symbol/broker;`/backtest/top` 排名只认主品种行 + 附带跨品种健壮性摘要与明细 |
| web view | `run()` 透传 `cross_symbol` 复选框 |
| web 模板 | 回测页运行表单加两个复选框(全券商禁用+提示);结果排名加"跨品种健壮性"列 + 展开明细行 |
| tables.js | 结果排名表 opt-in `data-detail-sort`:带详情行也能排序(排序只动主行,详情行按 toggle id 跟随粘合)——保住排序 + 展开两全 |

**排名纪律**:排名/晋级只认**主品种**结果(`backtests.symbol = strategies.symbol`),跨品种结果仅用于健壮性摘要与明细,不参与排名——避免"拿某个巧合品种的成绩去排名"。

---

## 5. 阈值进 config(不写死)

健壮性配色门槛(如"多数赚"= 60%)先用常量,后续需要收紧再挪进 config;#1 OOS 落地时,切分比例 / 留出段最少笔数 / 短样本月数门槛一并进 config。
