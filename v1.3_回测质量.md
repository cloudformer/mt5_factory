# v1.3 回测质量(反过拟合)

> 状态:**本期实现「货币对 + 券商」两个筛选(纯 filter)**;真正的反过拟合手段(OOS 时间切分、跨品种健壮性)见 §1,仍为规划。
> 目标:根治"回测显示很好、demo 完全不一样"——本质是**过拟合**(在短历史上挑中了靠运气的漂亮参数)。
> 边界:**只加回测的"筛选/质量",不碰撮合引擎**(撮合已足够悲观,见 §0)。策略逻辑、下单路径一律不动。

## 0. 先厘清:撮合是准的,漏的是"筛选"

`services/backtest.py` 的撮合已经足够悲观、足够真实,**不是问题所在**:

- 信号在 TF bar 收盘产生 → **下一根 M1 开盘价成交**(不偷看未来)
- 进场按 ask+点差+滑点,离场按 bid;**SL 先于 TP 检查**(同 bar 悲观)
- **跳空按实际开盘价成交**(sl_gap/tp_gap),不按挂单价
- 佣金按点数逐笔扣除;点差默认用每根 bar 记录的真实点差

所以"回测好 demo 差"**不是撮合失真**,而是:**没有反过拟合的筛选**——随机生成的一批参数里,总有几条在这段历史上碰巧漂亮;拿去 demo(=样本外)就现原形。

## 1. 反过拟合总方案(手段 / 治什么 / 代码改动风险 / 是否做)

过拟合有两个正交维度:

```
时间维度(#1 OOS):     同货币对, 过去 vs 未来(留出段)都得赚
空间维度(#5 跨品种):   同参数, 这个货币对 vs 别的货币对都得赚
两个都过 = 才够格进 demo
```

| # | 手段 | 治什么 | 代码改动 / 风险 | 是否做 |
|---|------|--------|----------------|--------|
| — | **货币对 / 券商 筛选**(纯 filter) | 不是反过拟合,是**易用性**:圈定回测/查看哪些货币对、哪个券商的策略 | 只加两个筛选维度,不碰撮合、不跨品种 | ✅ **本期已实现** |
| **1** | **样本外切分**(时间维度) | 过拟合主因(demo 就是样本外) | 回测跑完后按时间把 trades 切训练段/留出段,各算 metrics;排名先按训练段选、再要求留出段也赚。**纯后处理,撮合零改动** | 🔜 下一期(第一优先) |
| **2** | **分段最少笔数** | 小样本假象 | 训练段、留出段各自都要够笔数;`/backtest/top` 筛选加强。极低风险 | 🔜 跟 #1 一起 |
| **3** | **短样本警告** | 数据太浅放大一切过拟合(现仅~3个月) | 回测算 M1 时间跨度,不足门槛标"样本不足,结论不可靠"。低风险 | 🔜 跟 #1 一起 |
| **5** | **跨品种健壮性**(空间维度) | 品种特异性过拟合(只在 XAU 历史巧合赚) | 同参数在多个货币对各回测一次,产出"N/M 赚"健壮性 + 明细。需跨品种展开(不是本期的纯 filter) | 🔶 视需要(与 #1 正交) |
| **4** | **参数邻域稳健性** | 参数卡在历史噪声缝里(孤峰) | 对候选生成 ±10~20% 邻居批量回测,要求邻居也普遍赚。新编排,算量 ×N | 🔶 第二期 |
| **6** | **walk-forward** | OOS 的滚动升级(金标准) | 多窗口编排,大改 | ❌ 暂不做 |

> **注(本期取舍):** 一度实现过 #5(跨品种健壮性列),后与用户确认**当前只要纯 filter(甲)**、不要跨品种展开(乙),故 #5 已回退为规划。#5 与本期的"货币对筛选"不是一回事:筛选是"每个策略只跑自己的货币对,选看哪些";#5 是"同一参数在别的货币对上也跑,看是否普适"。

**数据深度不改代码,却是地基**:引擎不关心 3 个月还是 10 年,但样本越深、OOS 越可信。深度是"下载/存储"的事,与回测逻辑无关。

## 2. 本期实现:货币对 + 券商 两个筛选(纯 filter)

回测**按货币对进行**(每个策略跑它自己的货币对,一如既往)。本期在回测页加**两个筛选维度**,决定"回测/查看哪些策略":

```
批量回测  [状态] [上限] [策略ID]  货币对[全部▾]  券商[全部▾]  [成本…]  [启动]
结果排名  货币对[全部▾]  券商[全部▾]  [最少交易数]  [筛选]
```

- **货币对**:全部 = 不限;选具体货币对 = 只回测/只看该货币对的策略。
- **券商**:全部 = 不限;选具体券商 = 只回测/只看该券商的货币对(按 `symbols` 表的券商标签圈定品种)。
- 选定券商 + 货币对全部 = **该券商的所有货币对**。两个维度全按选项走。
- 下拉选项从**库里** group by 出来(货币对 = download 品种;券商 = 品种主档里出现过的券商标签),默认"全部"。

### 券商这个标签从哪来(关键,别再混)
- **只有"下载数据"跟 worker 打交道**:下载时把**当前券商**(worker 账户 server 名)写进 `symbols.broker`。这是券商唯一的落库时刻。
- **回测之后完全不碰 worker**:回测只读数据库,按库里已有的券商标签过滤,和 worker/在线状态无关。
- **原来**没有券商列 → 等价于"当前货币对 × 所有券商"(其实就一家、没打标、没这概念)。现在券商成了可选筛选维度。
- 今天只有一家券商 → 下拉就一个选项,选"全部"照跑;接了第二家自动多出来。
- 补充:同一货币对"多家券商价格序列并排比"是另一回事,需**重量版 schema**(broker 进 `historical_bars` 主键),与本期筛选无关。
