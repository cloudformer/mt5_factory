# v5.4 市场字段与展示区分(第四步 · 只加不变)

> 序列见 v5.1。给策略市场打数据地基 + 展示层按字段区分;放行/隐藏的执法仍在 5.6。

## 功能

1. strategies 补市场溯源字段:
```sql
ALTER TABLE strategies ADD COLUMN origin_id INTEGER REFERENCES strategies(id);
-- fork 来源: 从哪个公开策略复制来的(与 parent_id 不同 — parent 是AI调参谱系, origin 是市场复制)
```
2. 展示区分:策略列表加 owner / visibility 两列(筛选面板可按其过滤);
   市场页雏形(只读列表):public/shared 策略的 模板/品种/成绩摘要/实盘汇总。

## 可见性语义(现在定死, 5.6 按此执法)

| visibility | 参数 | 回测成绩 | 实盘汇总 | 别人能做什么 |
|---|---|---|---|---|
| private | 只有 owner | 只有 owner | 只有 owner | 什么都看不见(默认) |
| public | 全可见 | 全可见 | 全可见 | 看 + 复制(fork 成自己的 private, origin_id 溯源) |
| shared | **不可见** | 汇总可见 | 汇总可见 | 看汇总 + 盲测(标准回测, 输出只放行汇总视图) |

**盲测 = 标准回测,引擎零分叉(2026-07-22 与 Frank 定死)**:
同一个回测引擎、同一个数据源、全量计算、逐笔照常入库——策略/数据/回测结果对引擎完全通用,
不存在"盲测模式"。"盲"只发生在**输出/视图层**:视图已拆成
回测汇总/回测明细/实盘汇总/实盘明细/交叉比对,未来就是按可见性决定谁能切哪个视图——
shared 策略非 owner 只放行两个汇总视图,明细与交叉视图不放行。

**防逆向红线(写死, 执法在输出层)**:shared 对非 owner **绝不返回逐笔明细**——
逐笔入出场时间价格≈把参数递给人反推;是 api 响应按可见性裁剪字段,不是前端藏。
实盘汇总来自 strategy_stats 聚合,不露账户/券商敏感信息。

## 本步边界

- 现在只有你一个用户,市场页=你自己给策略打标后的预览(检验语义顺不顺手);
- fork/盲测两个**动作**涉及归属写入与参数保护,放 5.6 之后随隔离一起上。

## 风险 / 难度 / 验收

- 风险:**无**(加一列+只读页面);难度:低-中(市场页雏形);
- 验收:全站回归无差异;打标 public/shared 的策略出现在市场页,private 不出现。
