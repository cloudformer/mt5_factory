# v2.1 AI 闭环 · 第一件:单策略 AI 成绩单(已上线 2026-07-17)

## 定位

AI 调参闭环的第一环:**把单个策略实例的全部事实打包成一份结构化 JSON**,喂给 AI 生成器做参数迭代。

```
成绩单 JSON ──→ AI("这是全部事实, 给下一组参数") ──→ 新参数 ──→ 回测 ──→ 新成绩单 ──→ 循环
     ↑                                                                        │
     └────────────────── 实盘/对账数据随 worker 运行持续更新 ←─────────────────┘
```

## 入口

- API:`GET /strategies/{id}/report`(api:8010)
- Web 透传:`/strategies/{id}/report.json` — 策略分析页「AI成绩单 JSON」按钮
- 纯数字无评语:**事实只存一份,表述现算**(页面提示语与本 JSON 同源渲染,不单独存)

## 详细度原则(2026-07-16 定)

单策略深分析是低频、单条的闭环动作 → **详细度优先,不截断不省略**。
唯一例外:**交叉品种只带成绩汇总**——交叉的角色是及格线筛查,交叉不灵就不会深分析,逐笔属于过度供给。

## 层级

| 顶层键 | 内容 | 粒度 |
|---|---|---|
| `strategy` | id/名/模板/参数/品种/周期/状态/magic/死因 | 全 |
| `backtests[]` 主品种 | metrics(OOS 训练/留出 · by_year 分年 · mae_p90/mfe_p90 · 成本假设) + `trades` 全量逐笔 9 列:`entry_time, exit_time, dir, entry, exit, points, reason, mae, mfe` | **全量逐笔** |
| `backtests[]` 交叉品种 | 仅 metrics 汇总(笔数/净点/胜率/PF/回撤) | 汇总 |
| `attribution_backtest` | 主品种回测归因聚合:by_hour/by_weekday/direction/reasons + summary(连亏/单笔集中度) | 聚合 |
| `attribution_actual` | 实盘归因聚合(与上同构)+ money 折合金额 + envs 来源 | 聚合 |
| `trades_actual` | 实盘全量逐笔 16 列:`entry_time, exit_time, dir, volume, entry_price, exit_price, sl, tp, net_points, profit, commission, swap, reason, env, broker, account` | **全量逐笔** |
| `reconciliation` | 对账**现算**(compute_reconcile 与分析页共用):匹配率 4 项 + v1 判定 + 精度偏差(entry_diff_avg/net_diff_avg/net_bias_pct, 正=回测乐观) + mode(segments/one_sided) + `windows[]` 对比窗口 + `pairs[]` 逐笔对照(window=归属窗口号, 缺口三分类 gap) | 全 |
| `runtime[]` | 运行区间原始段(from/to/host)— 策略何时真实在跑 | 全 |
| `actual` | 实盘总账 + `by_env`(DEMO/LIVE 各自笔数/胜负/盈亏) | 汇总 |
| `failed_neighbors[]` | 同模板已淘汰的参数 + died_of 死因码(负样本, ≤20 条) | 汇总 |

逐笔用列式数组(`{cols:[], rows:[[]]}`)——格式紧凑,内容零删减。

## AI 拿它能回答的问题(设计验证, 以 #706 实测为例)

- **方向不对称**:SELL 61 笔 +28688 vs BUY 58 笔 +6926 → 考虑只做空/降多头权重
- **时段结构**:12-16 点与 21-22 点合计约 -25000, 盈利集中亚洲盘 → 时段过滤
- **出场结构**:77 笔 SL(-116017) vs 40 笔 TP(+149486) → 低胜率高赔率型;配 MAE 分布可算"SL 收多少砍输单不伤赢单"
- **可信度背书**:对账 100%、总账偏 +0.7%、留出段 +11002 → 上述结论站在可信回测上
- **负样本**:failed_neighbors 的参数区域 → 不再生成同类

## 相关阈值/机制

- 交叉测试门槛 `cross_symbol_gate`(config, 021):批量回测只给主品种过线的展开交叉;按 ID 点名不设限;全空=全交叉
- 死因码 `strategies.archive_reason`(022):manual/holdout_loss/min_trades/low_pf/recon_fail/orphan_symbol/other;归档表单选码, 复活自动清空
- MAE/MFE:回测引擎逐笔记录(services/backtest.py), metrics 聚合 p90

## v2.2 设计存档:DSL 层(2026-07-17 讨论定稿, 代码未建 — AI 生成启动时照此实施)

**定位**:AI 安全生成的载体。策略生成分两轨:
- **量产轨(DSL)**:AI 产出受限 JSON(非任意代码), 机器校验, 全自动进漏斗 — 覆盖 ~90% 结构组合搜索
- **精品轨(Python 模板)**:AI 写完整模板代码, 走门禁(语法→冒烟→回测→**人工 git 审查**)— 复杂/带状态策略

**落地形态(关键决定)**:不做"文本 DSL + 代码生成", 做 **JSON 规格 + 解释器模板**——
解释器是 strategy_core 第 4 个模板(进 git), 每份规格是它的一个实例(params=规格 JSON, 进 DB)。
零代码生成、零新文件、回测/runner/漏斗/对账/成绩单零改动。"参数进DB代码进git"纪律不破。

**表达力 v1 封顶**(边界越小 AI 越安全; 可控地逐词扩):恰好覆盖现有三模板 —
指标 `close/sma/ema/rsi/atr/highest/lowest`, 条件 `>/</>=/<=/cross_above/cross_below` + `all` 组合,
出场 `points/atr` 两种 SL × rr。带状态逻辑(移动止损/加仓)/多周期/自定义数学 = 精品轨, 不塞进 DSL。
候选扩词:时段过滤(hour), 多周期操作数(tf), any 组合。

**规格骨架**:
`{"dsl":1, "long":{"all":[{"lhs":{"ind":"sma","n":12},"op":"cross_above","rhs":{"ind":"sma","n":65}}]},`
`"short":{...}, "exit":{"sl":{"mode":"points","value":1500} 或 {"mode":"atr","period":14,"mult":2.0}, "rr":2.5}}`
cross 语义 = prev(lhs)≤prev(rhs) 且 now(lhs)>now(rhs); ema 用窗口化确定性实现(4n 窗+前 n 均值种子, 无状态)。

**验收标准**:用 DSL 复刻三个手写模板, 同数据回测逐 bar 信号一致(解释器正确性自证)。
**MQL5**:同一份规格可再做"规格→.mq5 生成器"(那侧必须代码生成, MT5 不解释 JSON);
回测永远跑 Python 解释器(关1 不需要 MQL5), EA 语义等价靠现有关2 对账验(magic 回流即生效)。
**工期评估**:DSL+解释器+创建接口+冒烟 ≈ 1 周; MQL5 生成器(到"可编译 EA+可信回测")再 +1.5~2 周。
