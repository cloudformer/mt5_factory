# v9999 迷宫游戏(一次性 · Python 亲子学习项目)

> 目标:带孩子学 Python。自动生成迷宫 → 验证有路 → 评难度 → 手动玩 → 各自写 bot 比赛。
> 原则一:**一步一功能**——每次只做一个功能,做完验收,且**不许破坏已验收的功能**(铁律)。
> 原则二:**简洁**——起步小、注释多、目录专业;最终不怕长大变复杂,但每一步必须小。
> 本文档自带每阶段的 AI 提示词,直接复制粘贴即可开工。

## 零、环境准备(第一课,只做一次)

用虚拟环境 venv——项目的包装在自己家里,不污染系统 Python(顺便学会所有 Python 项目的标准开局):

```bash
mkdir maze_game && cd maze_game
python3 -m venv .venv                 # 创建虚拟环境(就是个目录)
source .venv/bin/activate             # 进入(Mac/Linux; Windows: .venv\Scripts\activate)
pip install pygame                    # 装在 .venv 里, 不碰系统
pip freeze > requirements.txt         # 记录依赖, 别人一条命令还原
```

以后每次开工先 `source .venv/bin/activate`(命令行前面出现 `(.venv)` 就对了)。
`.gitignore` 写三行:`.venv/`、`__pycache__/`、`maze.db`(环境和数据不进 git,代码才进)。

## 〇.五、开发纪律(全程有效,比任何功能都重要)

1. **一步一功能**:一次只让 AI 写一个功能,跑通、验收、commit,再要下一个;
2. **不破坏旧功能**:每完成一步,把**之前所有验收命令重跑一遍**(维护一个"验收清单",
   做一步追加一条)——旧命令有一条坏了,先修回去再继续。这就是最朴素的回归测试;
3. **git 每步一提交**:`git commit -m "阶段1: 迷宫生成"`,改坏了随时能回去。

## 二、数据库(SQLite, 两张表, 全关联)

```sql
-- 地图: 生成即入库, 同一张图可反复玩/反复跑bot(公平比较的基础)
CREATE TABLE maps (
    id         INTEGER PRIMARY KEY,
    width      INTEGER NOT NULL,
    height     INTEGER NOT NULL,
    grid       TEXT    NOT NULL,   -- 字符画: '#'墙 '.'路 'S'起点 'E'终点, 行用\n分隔
    shortest   INTEGER NOT NULL,   -- BFS 最短路步数(验证有路的副产品)
    difficulty REAL    NOT NULL,   -- 难度分 1~10
    created_at TEXT DEFAULT (datetime('now'))
);

-- 对局: 人玩一局/bot跑一局都是一行, 挂 map_id 外键 → 人和bot在同一张图上可比
CREATE TABLE plays (
    id         INTEGER PRIMARY KEY,
    map_id     INTEGER NOT NULL REFERENCES maps(id),
    player     TEXT    NOT NULL,   -- 'human:爸爸' / 'bot:RightHand' / 'bot:小明1号'
    steps      INTEGER NOT NULL,   -- 走了多少步
    success    INTEGER NOT NULL,   -- 1=到终点 0=放弃/超步数上限
    created_at TEXT DEFAULT (datetime('now'))
);
```

比赛就是一句 JOIN:同一 `map_id` 下按 `steps` 排序,谁少谁赢。

## 三、核心算法(都是入门经典,顺序就是学习顺序)

| 功能 | 算法 | 教学点 |
|---|---|---|
| 生成迷宫 | **递归回溯挖墙**(DFS):从起点出发,随机选方向挖,走不动就退回上一格 | 递归!和"递归遍历文件目录"一模一样的思路——进目录=挖新格,目录到底=回退 |
| 验证有路 | **BFS 广度优先**:从 S 一圈圈泛洪,能碰到 E 就有路,顺便得到最短步数 | 队列、循环、visited 集合 |
| 难度评分 | 公式:`难度 = 最短路步数/(宽+高) × 4 + 死胡同数/总路格 × 6`,截到 1~10 | 简单算术 + 遍历统计(count) |
| 手动玩 | pygame 画格子 + 方向键移动,撞墙不动,到 E 结算入库 | 事件循环、坐标 |
| bot | 每回合给 bot 看四邻(上/下/左/右是墙还是路),bot 返回一步 | 类与继承、状态记忆、计数 |

## 四、bot 设计(第 3 阶段的核心玩法)

**接口只有一个函数**,孩子改造零门槛:

```python
class Bot:
    name = "基类"
    def __init__(self):
        self.memory = {}            # 想记什么自己放(走过的格子/计数器…)
    def next_move(self, pos, walls):
        """pos=(x,y) 当前位置; walls={'U':True,...} True=那边是墙。
        返回 'U'/'D'/'L'/'R' 中的一个。每回合只看得到四邻——和真人一样"""
        raise NotImplementedError
```

自带 3 个示例,由浅入深,孩子照着抄第 4 个:

1. `RandomBot` —— 随机选一个不是墙的方向(几行代码,先跑通流程);
2. `RightHandBot` —— 右手扶墙法(经典:一直贴着右边墙走,简单迷宫必出);
3. `CountBot` —— 记住每格来过几次(`memory` 里计数),优先走次数最少的邻格——**天然用上循环和 count**,而且真的聪明。

**竞技场规则**(`arena.py`):同一批图(比如 10 张,从库里按 id 取,保证公平)每个 bot 都跑,步数上限 = 最短路 × 10(防死循环)。结果入 plays 表,打印:

```
地图#3(难度6.2, 最短38步): 小明1号 52步 ✓ | RightHand 88步 ✓ | Random 380步 ✗超时
总榜: 小明1号 平均1.4倍最短路 > RightHand 2.3倍 > ...
```

## 六、边界与长大规则

三阶段内不做:存档读档、关卡皮肤、音效、动画、网络对战、AI 寻路以外的算法炫技、ORM。
数据库先两张表;pygame 只画矩形和圆。

**三阶段之后不设天花板**——想加什么都行(对角线迷宫?bot 视野变远?难度公式改进?
排行榜页面?),但守住同一条纪律:**一次一个功能、旧验收清单全绿才算完成**。
项目可以慢慢长得很复杂,过程永远是小步。
