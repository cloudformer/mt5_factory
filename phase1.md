> 之后日常只做策略工作(生成/回测/筛选/晋级),不再碰基建。
> 组合/桶/手数等规划见 STRATEGY.md(第二阶段)。

- [x] web 生成策略实例(模板×参数×品种×周期)→ strategies 表
- [x] 状态流转 CANDIDATE→DEMO→LIVE→ARCHIVED,任意可互转,进 DEMO/LIVE 自动分配 magic

- [x] 每策略独立 magic,同策略同时最多 1 仓

## 3. 停止 / 暂停 ✅

- [x] 取消主机指派 → runner 60s 内自动停止交易(自愈,不用登 Windows)
- [x] 策略移出 DEMO/LIVE → 该策略停止开仓
- [x] 停用主机 → 心跳/下载/交易全部排除
- [x] 停止只停开仓;已有持仓由服务端 SL/TP 兜底,不产生风险窗口

## 4. 可靠性 ✅

- [x] runner 无状态:重启后从 MT5 读持仓、从 DB 读配置,零恢复成本
- [x] 当前模板退出只有 SL/TP(挂券商服务器)→ worker 掉线**不存在错过平仓**
- [x] bridge 自愈:终端僵死连续 6 次附着失败自动杀掉重拉
- [x] 看门狗 + 开机自启;单策略异常隔离,不拖累其他策略
- [x] 心跳三态:在线 / MT5未就绪(bridge 可达,可远程下发账户)/ 离线
- [x] worker 回传账户+每策略战绩,web 约 30s 一刷;爆仓风险面板(保证金水平)

## 5. 无状态接管(worker 掉线 → 克隆机顶上)✅ 设计已支持

标准顺序(防旧机复活双跑;克隆机 IP 不同,未指派前不会交易):

```
1. Demo/Live 页:取消旧机指派   ← 必须第一步
2. Workers 页:停用旧机
3. 克隆机开机 → 自动注册出现在 Workers 页
4. 下发同一 MT5 账户 → 变"在线"
5. Demo/Live 页:指派新机 → 1 分钟内自动接管
```

全程持仓由服务端 SL/TP 保护。
