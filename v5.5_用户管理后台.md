# v5.5 用户管理后台(第五步 · 只加不变)

> 序列见 v5.1。把用户运营动作页面化——机制是 5.2/5.3 的表,这里只是操作台。

## 功能(一个管理页)

- 用户列表:名字/启停状态/策略数/worker 数;
- 建用户 / 停用启用(users.enabled);
- 签发 api key(明文只显示一次)/ 吊销单把;
- **配置管理**:看该用户的 user_config 覆盖行(含 quota_* 额度),改/删(删=回全局默认);
- worker 归属调整(把某台 host 划给某用户)。

## 数据

无新表——全部复用 5.1~5.3。可选:操作留痕 `user_events`
(仿 mt5_host_events:user_id/event/detail/created_at),记"谁何时被停用/换key/改额度"。

## 权限口径(本步的临时态)

此时认证还没接(5.6 才接),管理页和全站一样"谁能访问网络就能用"——
**与现状的安全水位一致**(内网单人),不新增风险,但也别在 5.6 之前暴露到公网。
5.6 落地后本页只认超管(user 1 或 users.is_admin)。

## 风险 / 难度 / 验收

- 风险:**无新增**(见上,与现状同水位);难度:低-中;
- 验收:页面完成 建用户→发key→写/删配置覆盖→划worker 全流程,不再需要手敲 SQL。
