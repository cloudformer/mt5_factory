# v1.3 策略列表排序分页下推 SQL 【已完成 2026-07-19】

> 定位:架构体检第 2 条。已实现:排序/分页/评分全下推 SQL,只搬当页行;
> total 用窗口 count(*)。排名百分位(含并列取平均名次)用 rank()/count() 窗口复刻,
> **60 合成策略临时表对拍新旧分数逐一相等**后删除 Python 版打分函数。
> 后续加速(可选):v2.0.1 热字段提真列+索引后 ORDER BY 免拆包再快一级。

## 现状(问题)

`/backtest/top`:SQL 把**所有**符合条件的行搬回 Python → Python 排序 → 切出当前页。
翻页确实是重新请求,但每次请求仍是"全量搬+全量排",页码只决定切哪一段。

## 目标(Frank 确认的方向)

翻页时每页**现查**:排序和切页都在数据库完成,只搬回当前页的行——

```sql
ORDER BY (b.metrics->>'net_points')::float DESC NULLS LAST
LIMIT $page_size OFFSET ($page-1)*$page_size
```

- 默认排序(净点)/各过滤条件:直接下推,轻改;
- **排名模板评分**(四维百分位加权)是全集运算,下推用 SQL 窗口函数
  (`percent_rank() OVER (ORDER BY ...)` 四列加权)——工作量主要在这;
- `total` 用 `count(*)` 单独一句(或窗口 `count(*) OVER ()`)。

## 验收

- 万级策略下翻页接口耗时不随总量线性涨;
- 默认排序/评分排序/全部过滤条件结果与现版一致(切换前后对拍一遍);
- 前端零改动(接口出入参不变)。
