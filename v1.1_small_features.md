## 1. 现在可以做 ✅(本地即可开发)

### 1.1 结果排名 → 升级为「排名模板」加权评分 — 已完成(2026-07-14)
- **模板 = 四维权重 + 默认最少笔数**,存 config `ranking_templates`(四件套),**配置页可增删改名调权,不改代码**。
- 四维:稳定=PF / 盈利=净点数 / 风险=回撤小 / 健壮=跨品种盈利比(未跨品种=0分);各维先归一成 **0~100 排名百分位**(消除量纲)再加权(权重按比例归一,不必和=100)。
- 种子:稳定性(50/10/20/20, 300笔)、盈利(10/50/10/30, 100笔)、风险(30/10/40/10, 200笔)。
- 排名页:「排序」下拉(默认净点数 + 模板们),选模板自动带其最少笔数(可手改),表格多「评分」列(0~100)。
- 数据库规划:配置类小数据(整读整写/量少/不被join)→ config JSONB;若将来评分快照落库或被外键引用 → 编号 SQL 升独立表。
- 验证:评分数学(容器内真函数+假数据:高PF低回撤策略在稳定性模板 90 分居首、并列取平均、None=0)、UI 增/删/存全链路、页面/冒烟 ALL PASS。

### 1.2 策略列表 / 策略回测 去含糊 — 已完成(2026-07-14)
- **策略列表排名**(原策略列表, 导航改名):唯一工作台 — 全部策略一张表(身份+成绩+评分+健壮性),
  **未回测的也出现**(成绩"—"沉底; api 改为 strategies LEFT JOIN backtests);筛选(状态/品种/券商/多条件)/搜索/排名模板全在这。
- **参数/magic 挪进展开明细**(与跨品种明细同一个展开框), 主行不爆宽;未回测的展开提示"按ID填 N 去单测"。
- **策略回测**(纯执行):只剩 孤儿警告 + 批量回测(按筛选/按ID=单策略)+ 进度;结果排名整段删除。
- 过滤语义对未回测严谨:min_trades>0 或任何成绩门槛 → 未回测不通过;模板评分中未回测各维=0分。
