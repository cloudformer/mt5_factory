# v5 总览与工作量评估(参考用主表)

> 只做索引 + 风险/工作量评估,不含设计内容(设计各在对应分册)。
> 2026-07-22 评估,供排期参考。**风险**=会不会搞坏现有系统;**难度**=纯工作量。

## 全序列

```
v5.0 worker拓扑与策略挂载   ← 地基, 可独立于多用户先上
v5.1 策略归属字段
v5.2 用户体系与worker归属
v5.3 license项进配置
v5.4 市场字段与区分
v5.5 用户管理后台
v5.6 隔离与权限执法          ← 命门, 所有行为变更集中于此
v5.7 订阅执行                ← 变现闭环, 依赖 5.6
```

总纲(全 v5 受约束):不分叉逻辑,引擎单一代码路径对所有用户一致,
差异只在 **UI 视图 / 权限 / 配额** 三个加法薄层(详见 v5.1)。

## 三档归类(排期用)

- **真·零风险,可放心先铺**:v5.0-A、v5.1、v5.2、v5.3(纯加法,没人读新东西);
- **低风险但有工作量**:v5.4、v5.5(不碰现有逻辑,但是新页面/端点,费工不费险);
- **要小心验收的**:v5.0-B(碰实盘+统计+对账)、v5.6(命门)、v5.7(真金代跑)。

## 建议顺序与铁则

1. 先铺零风险层(5.0-A / 5.1 / 5.2 / 5.3),现状零影响,把字段/表/配置备好;
2. 再做低风险 UI(5.4 / 5.5),此时仍单用户,页面先长出来;
3. 最后集中攻 5.6(唯一命门)——**验收全绿前绝不发第二把 key**;
   验收=无 key 全 401 + 越权扫描全绿 + user1 全站回归无差异;
4. 5.0-B 按需插入(真要多账户/多 worker 分担时);5.7 在 5.6 立住后再上。
